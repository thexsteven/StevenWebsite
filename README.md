# Mein Portfolio 

Persönliche Portfolio-Website, um HTML, CSS und JavaScript zu lernen.

##  Ziel
Eine responsive Website über mich mit:
- Über-mich-Sektion
- Skills & Lebenslauf
- Projektgalerie
- Reiseberichte

##  Tech Stack
- HTML5 (semantisch)
- CSS3 (Flexbox, Grid)
- JavaScript (Vanilla)
- Git & GitHub Pages

##  Projektstruktur
- index.html
- css/styles.css
- js/
- images/
- assets/

##  Inhalte & Sektionen (geplant)
- Hero (Vorschau + Scroll-Einstieg)
- Über mich
- Lebenslauf (interaktive Timeline, seriöser Stil)
- Skills
- Karriere (Theorie- & Praxisphasen, Semesterzusammenfassungen)
- Reisen (Hauptfokus: KonstanzCannes, Hawaii  mit Videos)
- Sport (dynamischer Stil mit Motivationsmedien)
- Projekte (vorerst nur Titel + Beschreibung)
- Kontakt (E-Mail, LinkedIn, GitHub, Standort)

##  Design & UX Leitlinien
- Modern, verspielt, mit kleinen Easter Eggs
- Seriosität in Lebenslauf & Karriere-Sektion
- Farbschema: Dunkelblau (Emerson-ähnlich), Weiß, Grau, DHBW-Rot
- Typografie: Apple-Style (San-Francisco-ähnlich)
- Mobile-First: hohe Priorität
- Animationen: dezent, nicht ablenkend
- Dark Mode: später

##  Navigationsidee
- Sticky Navigation beim Scrollen
- Startseite als Vorschau der Inhalte
- Klickbare Kacheln/Bilder, die zu den Sektionen springen

##  Nächste Schritte (geplant)
1. Semantisches HTML-Grundgerüst
2. Basis-Designsystem (CSS-Variablen, Typografie, Spacing)
3. Sticky Navigation + Hero + Vorschau-Kacheln
4. Sektionen ausarbeiten (Inhalte + Layout)
5. Responsiveness & Feinschliff
6. Dezente Interaktionen & Easter Eggs

##  Git & Workflow (kurz)
- Kleine Schritte, jeder Schritt ein Commit
- Größere Änderungen auf Feature-Branches (z. B. feature/hero-section)
- main bleibt stabil
